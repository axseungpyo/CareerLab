# /bulk-review

자소서의 모든 항목(또는 전체 자소서)에 대해 4축 첨삭 분석을 일괄 수행합니다.
개별 점수, 공통 패턴, 우선순위 개선 목록을 제공하며, 선택적으로 수정안을 자동 적용할 수 있습니다.

## Prerequisites

- 백엔드 서버 실행 중 (`http://localhost:8000`)
- 최소 1개 자소서에 1개 이상의 항목이 저장되어 있어야 함
- Claude 서비스가 활성화되어 있어야 함

## Steps

### 1. 서버 상태 확인

`GET http://localhost:8000/api/settings/status`를 호출한다.
Claude 상태가 `valid`가 아니면 안내하고 중단한다.

### 2. 범위 선택

사용자에게 첨삭 범위를 확인한다:

```
첨삭 범위를 선택하세요:

1. 특정 자소서 — 하나의 자소서 전체 항목 분석
2. 전체 — 모든 자소서의 모든 항목 분석
```

#### 옵션 1: 특정 자소서
`GET http://localhost:8000/api/resume`으로 목록을 표시하고 선택받는다.
`GET http://localhost:8000/api/resume/{resume_id}`로 해당 자소서의 항목을 가져온다.

#### 옵션 2: 전체
`GET http://localhost:8000/api/resume`으로 모든 자소서를 가져온 뒤,
각 자소서에 대해 `GET http://localhost:8000/api/resume/{id}`로 항목을 수집한다.

### 3. 일괄 첨삭 분석

수집된 각 항목에 대해 순차적으로 첨삭 API를 호출한다:

```
POST http://localhost:8000/api/interview/review/analyze
Body: {
  "resume_item_id": "{item.id}",
  "question": "{item.question}",
  "answer": "{item.answer}"
}
```

각 호출 후 진행 상황을 표시:
```
[3/8] 카카오 - "지원동기" 분석 완료 (구조: 7, 내용: 5, 표현: 8, 전략: 6)
```

결과에서 추출할 데이터:
- `scores`: structure, content, expression, strategy (각 1-10)
- `analysis`: 축별 분석 텍스트
- `suggestions`: 개선 제안 배열 (type, original, revised, reason)
- `revised_text`: 전체 수정본
- `report_id`: 피드백 리포트 ID (적용 시 필요)

### 4. 점수 종합 테이블

모든 분석 완료 후 종합 테이블을 출력한다:

```markdown
## 첨삭 분석 결과

### 점수 테이블

| 자소서 | 문항 | 구조 | 내용 | 표현 | 전략 | 평균 |
|--------|------|------|------|------|------|------|
| 삼성전자 | 지원동기 | 8 | 7 | 8 | 7 | 7.5 |
| 삼성전자 | 직무역량 | 6 | 5 | 7 | 5 | 5.8 |
| 카카오 | 지원동기 | 7 | 8 | 6 | 7 | 7.0 |
| 카카오 | 성장과정 | 5 | 4 | 6 | 4 | 4.8 |  ← 최약

전체 평균: 구조 6.5 | 내용 6.0 | 표현 6.8 | 전략 5.8
```

점수 기준 표시:
- 8-10: 우수 (추가 수정 불필요)
- 5-7: 보통 (개선 여지 있음)
- 1-4: 미흡 (반드시 수정 필요)

### 5. 공통 패턴 분석

모든 첨삭 결과를 종합하여 공통 패턴을 찾는다:

```markdown
### 공통 패턴

**반복되는 문제:**
- 정량적 성과 부족: 8개 항목 중 5개에서 지적됨
- 진부한 표현 사용: "열정을 가지고" (2회), "노력하겠습니다" (3회)
- STAR 구조 불완전: Result 부분 누락 3건

**강점:**
- 구체적 프로젝트 사례 활용이 우수
- 기업 키워드 반영률 평균 70%
```

### 6. 우선순위 개선 목록

평균 점수가 낮은 항목부터 정렬하여 개선 리스트를 제공:

```markdown
### 우선순위 개선 목록

1. **[카카오 - 성장과정]** 평균 4.8
   - 내용: 구체적 사례 부족 → STAR 구조로 재구성 필요
   - 전략: 직무 키워드 "데이터 분석" 미포함
   - 수정안 미리보기: "대학 시절 데이터 분석 프로젝트에서..."

2. **[삼성전자 - 직무역량]** 평균 5.8
   - 내용: 정량적 성과 수치 없음
   - 구조: 결론 부분이 약함
   - 수정안 미리보기: "해당 프로젝트를 통해 처리 속도를 40% 개선..."

3. ...
```

### 7. 선택적 수정안 적용

사용자에게 적용 방식을 확인한다:

```
수정안을 적용하시겠습니까?

1. 전체 적용 — 모든 항목에 수정안 자동 적용 (버전 증가)
2. 선택 적용 — 항목별로 적용 여부 선택
3. 적용 안 함 — 결과만 확인
```

#### 전체 적용
각 report_id에 대해:
```
POST http://localhost:8000/api/interview/review/apply/{report_id}
```

#### 선택 적용
각 항목에 대해 원문 vs 수정안을 표시하고 적용 여부를 확인:
```
[카카오 - 성장과정]
원문 (247자): "저는 어릴 때부터 도전정신을 가지고..."
수정안 (312자): "대학교 3학년, 처음으로 데이터 분석 프로젝트를 주도하면서..."

적용? (y/n/diff):
```
`diff` 입력 시 상세 차이점을 표시한다.

### 8. 최종 요약

```markdown
## 일괄 첨삭 완료

- 분석 항목: 8개
- 적용된 수정: 5개 (버전 업데이트됨)
- 미적용: 3개
- 전체 평균 점수 변화: 5.8 → 7.2 (예상)

다음 단계:
- /resume-compare → 수정 전후 비교
- /interview-prep → 면접 준비 가이드 생성
- 웹 UI에서 확인: http://localhost:3000/review
```

## API Endpoints Used

| Method | Endpoint | 용도 |
|--------|----------|------|
| GET | `/api/settings/status` | 서비스 상태 확인 |
| GET | `/api/resume` | 자소서 목록 |
| GET | `/api/resume/{id}` | 자소서 상세 (항목 포함) |
| POST | `/api/interview/review/analyze` | 4축 첨삭 분석 |
| POST | `/api/interview/review/apply/{report_id}` | 수정안 적용 |

## Error Handling

- **항목 없음**: "이 자소서에 저장된 항목이 없습니다. 먼저 자소서를 생성하세요."
- **분석 실패**: 해당 항목을 건너뛰고 계속 진행. 실패 항목은 결과 테이블에 "[분석 실패]"로 표시.
- **적용 실패**: "수정안 적용에 실패했습니다. 수동으로 적용하려면: POST /api/interview/review/apply/{report_id}"
- **대량 호출 시 속도**: 항목이 10개 이상이면 "분석에 시간이 걸릴 수 있습니다 (항목당 약 5-10초)" 안내.
