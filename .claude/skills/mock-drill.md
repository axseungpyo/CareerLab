# /mock-drill

Claude Code CLI에서 모의면접을 진행하며, 웹 UI와 달리 **매 답변 후 즉시 코칭 피드백**을 제공합니다.
면접 종료 후에는 전체 평가 + 스터디 노트를 생성합니다.

## Prerequisites

- 백엔드 서버 실행 중 (`http://localhost:8000`)
- 최소 1개 자소서가 저장되어 있어야 함
- Claude 서비스가 활성화되어 있어야 함

## Steps

### 1. 서버 상태 확인

`GET http://localhost:8000/api/settings/status`를 호출한다.
Claude 상태가 `valid`가 아니면 안내하고 중단한다.

### 2. 자소서 선택

`GET http://localhost:8000/api/resume`을 호출하여 자소서 목록을 가져온다.
번호 목록으로 표시한다:

```
어떤 자소서로 면접을 준비할까요?

1. 삼성전자 자소서 (2024-03-10, 3문항)
2. 카카오 자소서 (2024-03-12, 2문항)
3. 네이버 자소서 (2024-03-14, 3문항)
```

사용자가 번호를 선택하면 해당 `resume_id`를 사용한다.

### 3. 모드 선택

```
면접 모드를 선택하세요:

1. normal — 친절하고 전문적인 면접관 (기본)
2. pressure — 압박 면접 (약점 파고들기, 반박, 구체적 근거 요구)
3. pt — PT 면접 (주제 발표 + 날카로운 질문)
```

기본값: `normal`

### 4. 질문 생성 확인

`GET http://localhost:8000/api/interview/questions/{resume_id}`를 호출한다.
기존 질문이 없으면:
```
POST http://localhost:8000/api/interview/generate-questions
Body: { "resume_id": "{resume_id}" }
```

생성된 질문을 카테고리별로 미리보기 표시:
```
예상 질문 생성 완료 (5개):
- [자소서 기반] 프로젝트 리드 경험에서 가장 어려웠던 점은?
- [직무 역량] 데이터 분석 도구 활용 경험을 설명해 주세요.
- [기업 적합] 왜 삼성전자에 지원하셨나요?
- [인성/가치관] 팀에서 갈등이 생겼을 때 어떻게 해결하나요?
- [돌발/압박] 경쟁사 대비 본인의 강점이 뭐라고 생각하세요?
```

### 5. 세션 시작

```
POST http://localhost:8000/api/interview/mock/start
Body: {
  "resume_id": "{resume_id}",
  "mode": "{선택한 모드}"
}
```

`session_id`를 저장한다.

다음과 같이 인터뷰 시작을 알린다:
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  모의면접 시작 ({모드} 모드)
  자소서: {기업명}
  종료하려면 "quit" 또는 "종료" 입력
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 6. 인터뷰 루프

사용자에게 답변을 입력받아 면접을 진행한다. 각 라운드:

#### 6a. 사용자 답변 전송
```
POST http://localhost:8000/api/interview/mock/chat
Body: {
  "session_id": "{session_id}",
  "content": "{사용자 답변}"
}
```
SSE 스트림을 소비하여 면접관 응답을 실시간 표시한다.

#### 6b. 답변별 즉시 코칭 (핵심 차별점)

면접관 응답을 표시한 **직후**, Claude Code 자체가 사용자의 답변을 분석하여 코칭을 제공한다.
면접관의 질문 맥락과 사용자의 답변을 기반으로 다음을 분석한다:

```
┌─ 코칭 피드백 ─────────────────────────────┐
│ 점수: 7/10                                 │
│                                            │
│ 강점: 구체적인 프로젝트 사례를 활용하여     │
│       설득력 있게 답변했습니다.              │
│                                            │
│ 개선점: 정량적 성과(숫자)를 추가하면 더      │
│         임팩트가 있습니다.                   │
│         "매출 15% 증가" 같은 구체적 수치를   │
│         포함해 보세요.                       │
│                                            │
│ 힌트: STAR 구조에서 Result 부분을 강화하면   │
│       이 질문에 대한 답변이 완성됩니다.       │
└────────────────────────────────────────────┘
```

코칭 분석 기준:
- **STAR 구조 충실도**: 상황-과제-행동-결과가 모두 포함되었는지
- **구체성**: 수치, 기간, 규모 등 정량적 요소가 있는지
- **질문 의도 대응**: 면접관이 묻고자 하는 핵심에 답했는지
- **톤 적절성**: 면접 모드에 맞는 톤인지 (pressure 모드에서 당황하지 않았는지)
- **키워드 활용**: 기업 분석에서 추출된 키워드를 자연스럽게 포함했는지

#### 6c. 계속 여부 확인

```
계속하시겠습니까? (Enter: 계속 / quit: 종료)
```

- "quit", "종료", "q" 입력 시 → Step 7로 이동
- 빈 입력 또는 다른 입력 → 다음 라운드 시작
- 5-7 라운드 후: "충분한 분량의 면접이 진행되었습니다. 계속할까요, 종료할까요?"

### 7. 세션 종료 및 평가

```
POST http://localhost:8000/api/interview/mock/end?session_id={session_id}
```

그 후 전체 평가를 요청한다:
```
POST http://localhost:8000/api/interview/evaluate/{session_id}
```

평가 결과를 구조화하여 표시:
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  면접 평가 결과
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

종합 점수: 72/100 (등급: B)

| 평가 축 | 점수 |
|---------|------|
| 답변 품질 | 7/10 |
| 논리성 | 8/10 |
| 전문성 | 6/10 |
| 커뮤니케이션 | 7/10 |
| 적응력 | 6/10 |

강점:
- 프로젝트 경험을 구체적으로 설명
- 논리적 구조가 명확

약점:
- 정량적 성과 제시 부족
- 압박 질문에서 당황하는 경향

종합 피드백:
전반적으로 ...
```

### 8. 스터디 노트 생성

평가 결과에서 `answer_feedback` 배열을 기반으로 스터디 노트를 생성한다:

```markdown
## 면접 스터디 노트: {기업명} ({날짜})

### Q1: {면접관 질문}
**내 답변 요약**: {답변 요약}
**점수**: 7/10
**피드백**: {피드백}
**모범 답변**: {모범 답변}
**개선 포인트**: {구체적 개선 사항}

---

### Q2: {면접관 질문}
...

---

### 핵심 개선 사항 (우선순위)
1. 정량적 성과 추가 연습
2. STAR 구조의 Result 부분 강화
3. 압박 질문 대응 전략 수립
```

사용자에게 스터디 노트를 표시하고, 필요 시 파일로 저장할 수 있음을 안내한다.

## API Endpoints Used

| Method | Endpoint | 용도 |
|--------|----------|------|
| GET | `/api/settings/status` | 서비스 상태 확인 |
| GET | `/api/resume` | 자소서 목록 |
| GET | `/api/interview/questions/{resume_id}` | 기존 질문 조회 |
| POST | `/api/interview/generate-questions` | 질문 생성 |
| POST | `/api/interview/mock/start` | 세션 시작 |
| POST | `/api/interview/mock/chat` | 면접 대화 (SSE) |
| POST | `/api/interview/mock/end` | 세션 종료 |
| POST | `/api/interview/evaluate/{session_id}` | 전체 평가 |

## Error Handling

- **자소서 없음**: "먼저 http://localhost:3000/resume/new 에서 자소서를 생성하세요. 또는 /batch-resume 으로 일괄 생성할 수 있습니다."
- **SSE 스트림 실패**: "면접관 응답 수신에 실패했습니다. 재시도합니다..." → 1회 재시도
- **평가 실패**: "평가 생성에 실패했습니다. 세션 ID ({id})로 나중에 다시 시도할 수 있습니다: POST /api/interview/evaluate/{id}"
- **사용자 중간 이탈**: "quit" 입력 시 현재까지의 대화로 평가를 진행할지 확인
