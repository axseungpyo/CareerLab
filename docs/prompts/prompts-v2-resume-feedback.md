# 자소서 생성 & 첨삭 프롬프트 v2.0

> `backend/config/prompts/resume_gen.yaml` + `feedback.yaml` 구조 설명 문서

## 1. 자소서 생성 프롬프트 (`resume_gen.yaml`)

### v1.0 → v2.0 변경점

| 항목 | v1.0 | v2.0 |
|------|------|------|
| **역할** | "자소서 컨설턴트" 1줄 | 15년 경력 대기업 코칭 전문가 + HR 관점 명시 |
| **작성 원칙** | 5줄 | 4대 핵심 원칙 + 5단계 작성 프로세스 |
| **문항 유형** | 없음 | 5가지 문항 유형별 전략 분기 |
| **금지 표현** | 2개 | 7개 + 대체 방향 |
| **입력 변수** | 4개 | 8개 (프로필 요약/자격역량/자소서 전략 추가) |

### System Prompt 구조

```
# 역할
15년 경력 대기업 자소서 컨설턴트

# 핵심 원칙 (4가지)
1. 채용담당자 관점 — "면접에서 만나보고 싶다"는 인상
2. 구체성 우선 — 추상적 다짐 < 수치화된 성과
3. 기업 맞춤 — 인재상·핵심가치·요구역량 매칭
4. 자연스러운 STAR — 기계적 나열 금지

# 작성 프로세스 (5단계)
1. 문항 읽고 평가 의도 파악 (직무역량/인성/조직적합성/문제해결력)
2. 매칭 경력에서 주력 소재 1~2개 선정
3. 기업 분석의 자소서 전략 반영
4. 추천 톤앤매너에 맞춰 작성
5. 서론(10%)→본론(75%)→마무리(15%) 비율 구성

# 문항 유형별 전략 (5가지)
- 지원동기/입사 후 포부 → 기업 이해도 + 직무 적합성
- 성장과정/영향 사건 → 가치관 형성 + 현재 역량 연결
- 사회이슈/시사 → 기업/산업 연결 + 균형 잡힌 시각
- 직무경험/강점 → STAR 깊이 + 수치화 성과
- 팀워크/갈등해결 → 구체적 상황 + 본인 역할 + 배운 점

# 금지 표현 (7개)
"열정을 가지고" / "노력하겠습니다" / "소통 능력이 뛰어나"
"도전정신" / "4차 산업혁명" / "귀사에 기여하고 싶습니다"
"학교에서 배운 것을 바탕으로"
```

### User Prompt 변수

| 변수 | 출처 | v1 | v2 |
|------|------|----|----|
| `company_analysis` | 기업 분석 JSON | O | O |
| `matched_entries` | pgvector 시맨틱 매칭 | O | O |
| `question` | 자소서 문항 | O | O |
| `tone` | 톤 선택 | O | O |
| `char_limit` | 글자수 제한 | O | O |
| `emphasis` | 강조 경력 | O | O |
| `resume_strategy` | **기업 분석 v2의 자소서 전략** | - | **NEW** |
| `profile_summary` | **프로필 요약 (학력/목표/핵심가치)** | - | **NEW** |
| `qualifications` | **어학/자격증/수상 요약** | - | **NEW** |
| `research_context` | 수동 리서치 파일 | O | O |

```

- 톤: {{ tone }}

```

## 2. 자소서 첨삭 프롬프트 (`feedback.yaml`)

### v1.0 → v2.0 변경점

| 항목 | v1.0 | v2.0 |
|------|------|------|
| **역할** | "첨삭 전문가" 1줄 | 15년 경력 + 채용담당자 관점 명시 |
| **4축 분석** | 각 1줄 설명 | 각 4~5개 세부 기준 + 8점 이상 기준 |
| **진부 표현** | 5개 | **15개** + 대체 방향 테이블 |
| **강점 분석** | 없음 | **strengths 배열 추가** |
| **코칭** | 없음 | **coaching 섹션 (전체 인상/최우선 개선점/면접 리스크)** |
| **진부 감지** | 없음 | **cliche_detected 배열** |
| **수정 우선순위** | 없음 | **suggestions에 priority 필드** |
| **입력 변수** | 3개 | 5개 (resume_strategy, char_limit 추가) |

### System Prompt — 4축 분석 기준 상세

#### 구조 (Structure) 1~10점
| 기준 | 설명 |
|------|------|
| 서론/본론/결론 비율 | 10%/75%/15% 권장 |
| STAR 충실도 | 상황→과제→행동→결과 자연스러운 연결 |
| 문항 의도 대응 | 묻는 것에 정확히 답했는가 |
| 단락 흐름 | 논리적 연결성 |
| **8점+ 기준** | 서론이 흥미를 끌고, 본론이 구체적이며, 마무리가 인상적 |

#### 내용 (Content) 1~10점
| 기준 | 설명 |
|------|------|
| 구체성 | 수치/사례/고유명사 포함 |
| 차별화 | 다른 지원자와 구분되는 고유 경험 |
| 기업 맞춤도 | 인재상/요구역량 매칭 |
| 깊이 | 표면 나열 vs 인사이트 |
| **8점+ 기준** | "이 사람만의 이야기"가 느껴지며, 역량을 구체적으로 증명 |

#### 표현 (Expression) 1~10점
| 기준 | 설명 |
|------|------|
| 간결성 | 불필요한 수식어/중복 제거 |
| 진부 감지 | 15개 표현 목록 체크 |
| 톤 일관성 | 시작~끝 동일 톤 |
| 능동태 | 수동태보다 능동태 사용 |
| **8점+ 기준** | 읽기 편하고, 진정성이 느껴지며, 기억에 남는 표현 1개+ |

#### 전략 (Strategy) 1~10점
| 기준 | 설명 |
|------|------|
| 키워드 포함도 | 채용공고 핵심 키워드 반영 |
| 면접 유도 | "더 듣고 싶다"는 부분 존재 |
| 약점 회피 | 공백기/전공 불일치 적절 처리 |
| 기업 연구 | 기업 사업/비전/문화 이해 |
| **8점+ 기준** | "이 지원자는 우리 회사를 잘 알고 있다"고 느낌 |

### 진부한 표현 감지 15개

| # | 진부한 표현 | 대체 방향 |
|---|-----------|----------|
| 1 | "열정을 가지고" | 열정이 드러나는 구체적 행동 사례 |
| 2 | "노력하겠습니다" | 구체적 실행 계획 (무엇을, 언제, 어떻게) |
| 3 | "도전정신" | 실제 도전한 사례 + 결과 |
| 4 | "소통을 잘하는" | 소통으로 문제를 해결한 구체적 상황 |
| 5 | "4차 산업혁명" | 구체적 기술명/트렌드 (AI, 클라우드 등) |
| 6 | "귀사에 기여하고 싶습니다" | 구체적 기여 방안 (어떤 프로젝트에 어떻게) |
| 7 | "학교에서 배운 것을" | 실제 프로젝트/과제에서 적용한 사례 |
| 8 | "항상 최선을 다해" | 특정 상황에서의 구체적 행동 |
| 9 | "글로벌 역량" | 해외 경험, 외국어 활용 사례 등 구체화 |
| 10 | "다양한 경험을 통해" | 핵심 경험 1~2개를 깊이 있게 |
| 11 | "꿈을 이루기 위해" | 구체적 커리어 목표 + 단계적 계획 |
| 12 | "팀워크를 발휘하여" | 팀에서 본인이 맡은 구체적 역할 |
| 13 | "많은 것을 배웠습니다" | 무엇을 배웠고 어떻게 적용했는지 |
| 14 | "성실하게 임했습니다" | 성실함이 드러나는 구체적 행동 |
| 15 | "어려움을 극복하고" | 어떤 어려움을, 어떻게 극복했는지 |

### v2.0 출력 JSON 구조

```json
{
  "scores": {
    "structure": 7,
    "content": 6,
    "expression": 8,
    "strategy": 5
  },
  "analysis": {
    "structure": "구조 분석 (잘한 점 + 개선점)",
    "content": "내용 분석 (잘한 점 + 개선점)",
    "expression": "표현 분석 (잘한 점 + 개선점)",
    "strategy": "전략 분석 (잘한 점 + 개선점)"
  },
  "strengths": [                              // ← NEW
    "이 자소서의 강점 1",
    "강점 2"
  ],
  "cliche_detected": [                        // ← NEW
    {
      "original": "원문에서 발견된 진부 표현",
      "suggestion": "대체 방향"
    }
  ],
  "suggestions": [
    {
      "type": "구조|내용|표현|전략",
      "priority": "high|medium|low",          // ← NEW
      "original": "원문 발췌",
      "revised": "수정안",
      "reason": "수정 이유"
    }
  ],
  "coaching": {                               // ← NEW
    "overall_impression": "전체 인상 (합격 가능성 관점)",
    "top_priority": "가장 먼저 개선해야 할 1가지",
    "interview_risk": "면접에서 취약점이 될 수 있는 부분"
  },
  "revised_text": "전체 수정본"
}
```

### v1.0 vs v2.0 출력 비교

| 필드 | v1.0 | v2.0 |
|------|------|------|
| `scores` | 4축 점수 | 동일 |
| `analysis` | 각 축 1줄 | 각 축 "잘한 점 + 개선점" 구조 |
| `strengths` | 없음 | **강점 배열** (동기부여) |
| `cliche_detected` | 없음 | **진부 표현 감지 결과** |
| `suggestions[].priority` | 없음 | **high/medium/low 우선순위** |
| `coaching` | 없음 | **합격 전략 코칭** (인상/최우선/면접리스크) |
| `revised_text` | 전체 수정본 | 동일 |

## 3. 데이터 흐름 (v2.0 전체)

```
사용자: 자소서 문항 입력
         │
         ├─→ [1] 기업 분석 v2.0 로드
         │     ├─ company_analysis (requirements, talent_profile, keywords)
         │     └─ resume_strategy (어필포인트/차별화/금지표현/톤/스토리텔링)  ← NEW
         │
         ├─→ [2] pgvector 시맨틱 검색 (경력 매칭)
         │     └─ matched_entries (유사도 순 경력 + STAR)
         │
         ├─→ [3] 프로필 확장 데이터 로드  ← NEW
         │     ├─ profile_summary (학력/목표/핵심가치)
         │     └─ qualifications (어학/자격증/수상)
         │
         ├─→ [4] 리서치 파일 로드 (선택)
         │     └─ ~/Documents/career/research/{기업명}.md
         │
         └─→ [5] Claude Sonnet 스트리밍 생성
               └─ 자소서 답변 (SSE)

사용자: 첨삭 요청
         │
         ├─→ [1] 자소서 문항 + 답변
         ├─→ [2] 기업 분석 (선택)
         ├─→ [3] 자소서 전략 (선택)  ← NEW
         │
         └─→ [4] Claude Sonnet 분석
               └─ JSON (scores + analysis + strengths + cliche + suggestions + coaching + revised)
```
