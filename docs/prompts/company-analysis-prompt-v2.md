# 기업 분석 프롬프트 v2.0

> `backend/config/prompts/company_analysis.yaml` 의 구조 설명 문서

---

## 1. 개요

| 항목 | 내용 |
|------|------|
| **버전** | 2.0.0 |
| **목적** | 채용공고 + 웹 리서치를 기반으로 **자소서 작성에 직접 활용 가능한** 전략적 기업 분석 |
| **모델** | GPT-4o-mini (json_mode) |
| **검색** | Tavily / Perplexity Sonar — 3카테고리 병렬 검색 |

---

## 2. 프롬프트 구조

### System Prompt — 역할 & 관점 정의

```
# 역할
당신은 10년 이상 경력의 한국 취업 컨설턴트이자 기업 분석 전문가입니다.
채용공고와 기업 정보를 분석하여, **자소서 작성에 직접 활용할 수 있는 전략적 인사이트**를 제공합니다.

# 분석 관점
- 채용담당자(HR)가 이 공고로 어떤 지원자를 원하는지 역추론
- 공고에 직접 명시된 것뿐 아니라 행간에 숨겨진 요구사항도 포착
- 기업의 산업 특성, 조직문화, 최근 동향이 채용 기준에 미치는 영향 분석
- 최종 목표: 지원자가 이 분석만 보고 설득력 있는 자소서를 작성할 수 있어야 함
```

**v1.0과의 차이:**
- v1.0: "당신은 한국 채용 시장 분석 전문가입니다" (1줄)
- v2.0: 역할(컨설턴트) + 목적(자소서 전략) + 분석 관점(HR 역추론, 행간 포착, 산업 맥락) 4줄

---

### User Prompt — 입력 데이터 구조

```
# 분석 대상
**기업명**: {{ company_name }}                    ← v2.0 신규: 기업명 직접 주입
**산업 분야 힌트**: {{ industry_hint }}            ← 선택적 산업 힌트

## 채용공고 원문
{{ job_posting_text }}

## 웹 리서치 결과                                  ← v2.0 신규: 3카테고리 구조화

### 기업 개요 및 문화
- **{{ title }}**: {{ description }}               ← 검색 쿼리: "{기업명} 기업문화 조직문화 핵심가치 비전 인재상"

### 최근 뉴스 및 동향
- **{{ title }}**: {{ description }}               ← 검색 쿼리: "{기업명} 최근 뉴스 사업 동향 실적 2025 2026"

### 채용 트렌드
- **{{ title }}**: {{ description }}               ← 검색 쿼리: "{기업명} 채용 면접 후기 합격 자소서 팁"

## 사용자 리서치 노트                               ← 선택적: 사용자 직접 입력
{{ research_notes }}
```

## 3. 출력 JSON 스키마

### 3-1. `company_overview` — 기업 개요

```json
{
  "industry": "산업 분야 (IT/소프트웨어, 금융, 제조, 유통 등)",
  "company_culture": "조직문화 특징 (자유로운/수평적/성과중심/안정지향 등)",
  "growth_stage": "기업 단계 (스타트업/성장기/대기업/글로벌 등)",
  "recent_focus": "최근 주력 사업 방향이나 화두"
}
```

**활용**: 상세 페이지 상단 4칸 그리드 카드로 표시

---

### 3-2. `requirements` — 채용 요구사항 (6분류)

```json
{
  "essential":            ["필수 — 없으면 탈락하는 조건"],
  "preferred":            ["우대 — 있으면 가산되는 조건"],
  "technical_skills":     ["기술 스킬 (프로그래밍, 도구, 자격증 등)"],
  "soft_skills":          ["소프트 스킬 (커뮤니케이션, 리더십, 협업 등)"],
  "experience":           ["경력/경험 요구사항 (연차, 도메인 경험 등)"],
  "hidden_requirements":  ["행간에서 읽히는 숨겨진 요구사항"]
}
```

**DB 저장 형태**: 카테고리 태그를 prefix로 붙여 플랫 배열로 저장
```
["[필수] Java/Spring 3년 이상", "[우대] MSA 경험", "[숨겨진] 빠른 성장 환경 적응력"]
```

**프론트 표시**: 카테고리별 컬러 태그
| 카테고리 | 색상 |
|----------|------|
| 필수 | 빨강 |
| 우대 | 파랑 |
| 기술 | 보라 |
| 소프트 | 초록 |
| 경험 | 황색 |
| 숨겨진 | 핑크 |

---

### 3-3. `talent_profile` — 인재상 (4차원)

```json
{
  "core_values":           ["기업이 중시하는 핵심 가치 (혁신, 도전, 고객중심 등)"],
  "ideal_candidate":       "이상적인 지원자 페르소나 (2~3문장)",
  "team_characteristics":  "예상되는 팀/조직 특성",
  "growth_opportunities":  "이 포지션에서 기대되는 성장 경로"
}
```

---

### 3-4. `keywords` — 핵심 키워드 (3분류)

```json
{
  "must_include":     ["자소서에 반드시 포함해야 할 핵심 키워드"],
  "domain_specific":  ["해당 산업/직무 특화 키워드"],
  "action_verbs":     ["임팩트 있는 동사 (주도하다, 개선하다, 달성하다 등)"]
}
```

**DB 저장**: 3개 카테고리를 합쳐서 `text[]` 배열로 저장

---

### 3-5. `resume_strategy` — 자소서 전략 (v2.0 핵심 신규)

```json
{
  "key_points":          ["자소서에서 반드시 어필해야 할 포인트 3~5개"],
  "differentiation":     "다른 지원자와 차별화할 수 있는 전략",
  "avoid":               ["자소서에서 피해야 할 표현이나 접근법"],
  "tone_recommendation": "추천 톤앤매너 (진솔한/전문적/열정적 등 + 이유)",
  "storytelling_angle":  "STAR 기법 활용 시 추천하는 스토리 방향"
}
```

**DB 저장**: `research_notes` TEXT 컬럼에 마크다운 형태로 변환 저장
```markdown
## 자소서 핵심 어필 포인트
- 데이터 기반 의사결정 경험 강조
- 팀 리딩 경험과 성과 수치화

## 차별화 전략
기술력 + 비즈니스 이해도를 동시에 보여줄 것

## 피해야 할 표현
- "열심히 하겠습니다" 같은 추상적 다짐
- 기업 홈페이지 문구 그대로 복사

## 추천 톤앤매너
전문적이면서 진솔한 톤. 이 기업은 실력주의이므로 구체적 수치와 사례 중심.

## 스토리텔링 방향
"문제 발견 → 데이터 분석 → 해결책 제시 → 측정 가능한 성과" 흐름
```

## 4. 데이터 흐름

```
사용자 입력
│  기업명: "네이버"
│  채용공고: "백엔드 개발자 채용..."
│  URL: "https://..."
│
├─→ [1] 웹 검색 (3카테고리 병렬)
│     ├─ 기업문화: "네이버 기업문화 조직문화 핵심가치 비전 인재상"
│     ├─ 뉴스:     "네이버 최근 뉴스 사업 동향 실적 2025 2026"
│     └─ 채용:     "네이버 채용 면접 후기 합격 자소서 팁"
│
├─→ [2] Jinja2 프롬프트 렌더링
│     ├─ system: 역할 + 분석 관점 + JSON 스키마
│     └─ user:   기업명 + 채용공고 + 3카테고리 검색결과
│
├─→ [3] GPT-4o-mini (json_mode=true)
│     └─ 7개 섹션 JSON 응답
│
└─→ [4] 후처리 & DB 저장
      ├─ requirements → [필수], [우대] 등 태그 prefix 부착 → jsonb
      ├─ talent_profile + company_overview → 병합 → jsonb
      ├─ keywords 3분류 → 플랫 배열 → text[]
      ├─ resume_strategy + interview_prep → 마크다운 변환 → research_notes
      └─ 검색결과 → category 태그 부착 → company_info jsonb
```

## 5. v1.0 → v2.0 비교표

| 항목 | v1.0 | v2.0 |
|------|------|------|
| **System 프롬프트** | 1줄 (전문가) | 역할 + 관점 + 목표 4줄 |
| **기업명 주입** | 없음 | `{{ company_name }}` 직접 전달 |
| **requirements** | `string[]` 플랫 | 6분류 (필수/우대/기술/소프트/경험/숨겨진) |
| **talent_profile** | `{핵심가치: [], 인재상: ""}` | 4차원 (가치/페르소나/팀특성/성장경로) |
| **keywords** | `string[]` 플랫 | 3분류 (필수포함/도메인/동사) |
| **자소서 전략** | `tips` 1줄 | 5항목 (어필포인트/차별화/금지표현/톤/스토리) |
| **면접 준비** | 없음 | 예상 주제 + 기업 특유 특징 |
| **기업 개요** | 없음 | 산업/문화/단계/주력방향 |
| **웹 검색** | 단일 쿼리 | 3카테고리 병렬 (문화/뉴스/채용) |
| **검색 결과 주입** | `"- title: desc"` 나열 | 카테고리별 구조화 마크다운 |
